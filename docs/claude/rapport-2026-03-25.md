## REVIEW SECURITE : MARKETPLACE AGENTS

### **#52 — CRITIQUE : IDOR sur `saveConversation`**

`conversationService.ts:65` — L'upsert utilise `where: { id: conversationId }` sans filtrer par `userId`. Le `conversationId` vient de `req.body` (client-controlled). Un attaquant peut envoyer le UUID d'une conversation d'un autre user et ecraser ses messages/status/metadata via la branche `update`.

```typescript
// ACTUEL (vulnerable)
await prisma.aIConversation.upsert({
  where: { id: conversationId },  // PAS de userId!
  ...
});
```

**Impact** : Un user peut lire/ecraser les conversations AI d'un autre user.

### #53 — HAUTE : `updateActiveStreamId` / `updateConversationStatus` sans userId

Meme pattern : `where: { id: conversationId }` sans guard userId. Combine avec #52, un attaquant peut manipuler l'etat de streaming d'une conversation tierce.

### #58-59 — MOYENNE : Rate limiting et limites de creation absents

- CRUD agents (`POST/PUT/DELETE /agents/custom`) sans rate limiter
- Pas de plafond sur le nombre d'agents custom par user (abuse de stockage)
- Seul `/generate-prompt` a un rate limiter (5 req/min)

### Points positifs du marketplace
- Auth middleware applique globalement (`authenticateToken` + `requireUser`)
- CRUD agents filtre correctement par `userId` + `isActive`
- System prompts des presets non exposes (`GET /presets` strip le champ)
- Validation Zod sur create/update agents
- Soft delete sur agents custom
- Pagination sur list endpoints (`take: 50`, `take: 20`)
- Timeout sur l'appel Gemini (`AbortSignal.timeout(60000)`)
- Sanitization des prompts custom (`sanitizeForPrompt` escape `<`/`>`)
